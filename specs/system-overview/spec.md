# 系统级目标与模块分层（草案）

## 目标
- 构建本地优先的个人 OS，逐步接入时间/精力、任务、笔记分析与可视化。
- 保持 Spec→Plan→Tasks 流程；跨模块协作用“架构桥”记录 RFC/Backlog。

## 模块层级
1) 数据采集/导出层
   - 日历/ActivityWatch/Mood（已在推进）。
   - Markdown/SQLite 笔记导出：全量 + 增量导出到本地目录（gitignore）。
2) 归一化与分析层
   - 时间/精力：日/周 JSON。
   - 笔记分析：AI/规则驱动，识别认知跃迁、里程碑等，输出结构化 JSON。
   - 任务管理：多维表格/提醒视图（规划中）。
3) 可视化层
   - 仪表盘：时间/精力/突发任务。
   - 笔记洞察：认知跃迁（方向感/判断力/分工能力/复盘能力）可视化。
   - 任务视图：多维表格、提醒事项。

## 新的 Futures（建议）
- FEAT-006-笔记导出与分析（新增目录）
  - 目标：从 SQLite/Markdown 全量导出 → 增量更新 → 本地存储（gitignore）。
  - 规范：明确导出目录、格式、增量策略。
- FEAT-007-认知跃迁识别与可视化（新增目录）
  - 目标：从本地 Markdown 批量识别认知跃迁/里程碑，输出结构化 JSON，并提供可视化设计。
  - 维度：方向感、判断力、分工能力、复盘能力。
- FEAT-008-任务管理与提醒（规划中）
  - 目标：多维表格/提醒视图，结合日历/任务/笔记信号。

## 数据与目录约定
- 导出目录（笔记/SQLite 等）必须 gitignore（例如 `data/notes_export/`），仅本地分析使用。
- 分析输出：结构化 JSON（如 `data/notes_insights/`），后续可视化读取。

## 监控领导力（认知跃迁）思路
- 数据源：Markdown 笔记（全量+增量）。
- 提取：AI/规则识别里程碑，映射到四维度（方向感、判断力、分工能力、复盘能力）。
- 输出：结构化 JSON（维度得分/标签/引用文档）；可视化仪表盘展示趋势与案例。
