# FEAT-006-笔记导出与增量策略（Bear/Markdown）

## 目标
- 从本地 SQLite/Markdown 全量导出笔记到 gitignore 目录，后续支持按标签、时间范围、增量更新的懒加载导出。
- 维护导出清单（manifest），避免重复导出，支持修改检测。

## 范围
- 数据源：Bear SQLite（数据库 + 附件），后续可扩展其他 Markdown 来源。
- 输出目录：`data/notes_export/`（gitignore），含正文/附件和 manifest。
- 模式：全量导出一次；后续按标签/时间范围/最近 N 天或自然语言指令做增量。
- 校验：通过唯一标识 + 更新时间或哈希判断是否已存在/需更新。

## 非范围
- 不上传导出内容到 GitHub。
- 不做云端同步，只做本地导出与分析。

## 验收
- 提供可运行脚本，支持：全量导出、按标签导出、按日期范围/最近一周导出。
- manifest 记录 note_id/更新时间/哈希/路径，重复执行不会生成重复文件，修改后会更新。
- 输出目录与 manifest 皆被 gitignore。
