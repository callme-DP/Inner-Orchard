# 时间与精力可视化（Future 任务）

## 背景
- 目标：以本地数据驱动的时间/精力监控，提供日/周/月/年视图的仪表盘（应用耗时、精力趋势、突发任务、日历事件概览）。
- 数据源：日历（可先用 Mock；按日历类别聚合）、ActivityWatch（应用/网页使用时长，后续接入）、MoodExample（早/午/晚情绪与精力）。

## 待办（Future Backlog）
- 接入真实日历数据：生成日/周/月/年 JSON，按日历类别聚合，确保“耗时=事件合计”。
- 接入 ActivityWatch：按 bucket 聚合应用类别耗时，支持本周 vs 上周对比。
- 精力趋势：读取 MoodExample 或本地 JSONL（早/午/晚 1-10 分），周视图可切换本周/上周；“全部”显示本周四条折线，单维显示本周 + 上周对比。
- 突发任务：定义 JSON 结构，按日统计并渲染（默认周视图，本周 vs 上周）。
- 日历事件表格：按日展示时间块、标题、类别、时长、触发/关联任务，随日/周/月/年切换联动。
- 渲染模板：在 `html/v1/` 放置基线模板，`html/output/` 生成最新仪表盘。
- 校验：生成前校验缺数据/缺权限并提示；生成后校验耗时对齐。

## 产出物
- `spec.md`：需求与范围。
- `quickstart.md`：快速上手，含数据准备与脚本调用示例。
- `tasks-guidelines.md`：任务生成规则（占位，测试策略待与宪章 4.4 对齐）。
